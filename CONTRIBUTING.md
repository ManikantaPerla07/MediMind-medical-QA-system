# Contributing

Thank you for your interest in improving MediMind.

## How to Contribute

1. Fork the repository.
2. Create a branch from main.
3. Keep each pull request focused on one change area.
4. Add or update documentation for behavior changes.
5. Open a pull request with a clear description.

## Local Development

1. Install backend dependencies from backend/requirements.txt.
2. Install frontend dependencies from frontend/package.json.
3. Build artifacts once using backend/startup.py.
4. Run backend and frontend in separate terminals.

## Pull Request Checklist

- The code runs locally.
- API behavior changes are documented in README.md.
- No secrets or private tokens are committed.
- Large generated files are excluded.
- Commit messages are clear and descriptive.

## Code Style

- Python: follow readable, consistent style and keep functions small.
- Frontend: follow existing component style and avoid unrelated refactors.
- Documentation: keep sections concise and actionable.

## Reporting Issues

When creating an issue, include:

- What you expected.
- What happened instead.
- Steps to reproduce.
- Relevant logs or screenshots.

## Security

Do not post credentials, tokens, or private health data in issues or pull requests.
